## 二、如何向 AI 提问更高效、更精准、更简洁

### 1. 结构化你的需求：「做什么 + 怎么做 + 约束条件」

| 维度 | 说明 | 示例 |
|------|------|------|
| **做什么** | 明确目标 | "新增社交焦虑测试分类" |
| **怎么做** | 技术路径（可选） | "用 Kotlin + Material3" |
| **约束条件** | 限制范围 | "25道题，4个选项，5级评估" |

**好的提问：**
> 在 TestData.kt 中新增「社交焦虑测试」分类，25道题，4选项（从不/偶尔/经常/总是），5级评估结果，同步更新 UI 和 MainActivity

**低效的提问：**
> 帮我加点东西

### 2. 一次说清，减少来回

把多个相关需求合并为一条消息，避免频繁 "Continue"。

**好的提问：**
> 完成以下3件事：
> 1. 新增AI谈心功能（测试结果后进入对话，根据分类匹配咨询师角色）
> 2. 版本号升级到1.1.0
> 3. 推送到GitHub并创建Release

**低效的提问：**
> 加个AI功能 → Continue → Continue → 推一下GitHub

### 3. 反馈问题时附上关键信息

报Bug时，附上**错误信息、链接、截图路径**，不要只说"不工作"。

**好的提问：**
> GitHub Actions 构建失败，错误日志：`AAPT: error: <adaptive-icon> elements require a sdk version of at least 26`，请修复

**低效的提问：**
> 这个上面没有编译呀

### 4. 用关键词标记优先级和范围

| 关键词 | 作用 |
|--------|------|
| `仅修改 xxx 文件` | 限定修改范围 |
| `不要改动 xxx` | 明确禁区 |
| `优先处理 xxx` | 设定优先级 |
| `完成后推送到GitHub` | 明确交付物 |

### 5. 常用高效提问模板

```
【功能开发】
在 [文件/模块] 中实现 [功能描述]，要求：
1. [具体要求1]
2. [具体要求2]
完成后 [后续动作：推送/测试/等]

【Bug修复】
[文件/链接] 出现 [错误描述]
错误信息：[粘贴错误日志]
请修复并重新 [构建/推送]

【重构/优化】
将 [当前实现] 改为 [目标实现]
影响范围：[列出相关文件]
注意：不要改动 [xxx]
```

## 三、总结

> **核心原则：说清「目标 + 范围 + 约束」，一次说完，附上关键上下文。**
>
> 你说得越精准，AI 执行得越快、越准确，来回越少。
